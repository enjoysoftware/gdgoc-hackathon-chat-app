# チャットアプリ セットアップガイド

この手順に従うことで、提供されたソースコードをベースに、リアルタイムチャット機能を有効化できます。

## 1. Firestore ドキュメントツリー設計

このアプリでは、ユーザー管理とチャンネルごとのメッセージ管理のため、以下の構造を採用しています。

### ユーザー情報
```text
/users (col)
  └ {userId} (doc) - Firebase Auth の UID
      ├ uid: string
      ├ displayName: string
      ├ photoURL: string
      ├ email: string
      ├ status: string      ("online" | "offline")
      └ updatedAt: timestamp
```

### チャンネル・メッセージ
```text
/channels (col)
  └ {channelId} (doc) - "project-alpha", "development" など
      └ /messages (col) - 各チャンネル固有のメッセージ
          └ {messageId} (doc)
              ├ text: string         (メッセージ本文)
              ├ senderId: string     (送信者のUID)
              ├ senderName: string   (送信時の表示名)
              ├ senderAvatar: string (送信時のアイコンURL)
              └ timestamp: serverTimestamp (送信日時)
```

## 2. Firebase プロジェクトの作成

1. [Firebase Console](https://console.firebase.google.com/) にアクセスし、「プロジェクトを追加」をクリックします。
2. プロジェクト名（例: `gdgoc-chat-app`）を入力し、作成を完了させます（Google アナリティクスは任意）。

## 3. Firestore Database の有効化

1. コンソール左メニューの「構築」>「Firestore Database」を選択します。
2. 「データベースの作成」をクリックします。
3. ロケーションを選択し、「テストモードで開始」を選択して「作成」をクリックします。

## 4. Firebase Authentication のセットアップ（Google ログイン）

このアプリは Google ログインを使用します。以下の手順で有効化してください。

1.  コンソール左メニューの「構築」>「Authentication」を選択します。
2.  「始める」ボタンをクリックします。
3.  **「Sign-in method」タブ** を開き、「新しいプロバイダを追加」から **「Google」** を選択します。
4.  右上のスイッチを **「有効にする」** に切り替えます。
5.  **「プロジェクトのサポートメール」** を自分のメールアドレスに設定し、「保存」をクリックします。
6.  （任意）ローカル開発以外で利用する場合は、「設定」タブの「承認済みドメイン」にドメインを追加する必要があります（`localhost` はデフォルトで許可されています）。

## 5. Firebase 構成情報の取得

1. プロジェクトの概要ページで、中央の **ウェブアイコン (</>)** をクリックしてアプリを登録します。
2. アプリのニックネームを入力し「アプリを登録」をクリックします。
3. 表示された `firebaseConfig` オブジェクトの内容をメモします（後で `.env.local` に使用します）。

## 6. ローカル環境の設定

プロジェクトのルートディレクトリで以下の操作を行います。

### 1. 依存ライブラリのインストール
ターミナルで以下を実行します。
```bash
npm install
```

### 2. 環境変数の設定
ルートディレクトリに `.env.local` ファイルを作成し、手順5で取得した値を貼り付けます。

```env
NEXT_PUBLIC_FIREBASE_API_KEY=あなたのAPIキー
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=あなたのプロジェクトID.firebaseapp.com
NEXT_PUBLIC_FIREBASE_PROJECT_ID=あなたのプロジェクトID
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=あなたのプロジェクトID.appspot.com
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=あなたの送信者ID
NEXT_PUBLIC_FIREBASE_APP_ID=あなたのアプリID
GEMINI_API_KEY=あなたのGemini APIキー(AI Studioで作成)
```

## 7. アプリの起動と確認

1. 以下のコマンドで開発サーバーを起動します。
   ```bash
   npm run dev
   ```
2. ブラウザで `http://localhost:3000` を開きます。
3. 右上の「Login with Google」からログインします。
4. 下部の入力エリアにメッセージを入力し、送信できることを確認してください。
5. Firebase Console の Firestore 画面を開き、`users` コレクションと `channels/project-alpha/messages` コレクションにデータが保存されていることを確認してください。

---

### 補足：セキュリティルールについて（推奨）
「テストモード」の期限が切れる前、または本番運用を意識する場合は、Firestore の「ルール」タブを以下のように更新してください。

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // ユーザー情報のルール
    match /users/{userId} {
      allow read: if true;
      allow write: if request.auth != null && request.auth.uid == userId;
    }
    
    // チャンネルとメッセージのルール
    match /channels/{channelId}/messages/{messageId} {
      allow read: if true; // 誰でも閲覧可能
      allow create: if request.auth != null; // ログイン済みなら作成可能
      allow update, delete: if false; // 基本的に変更・削除は不可
    }
  }
}
```
