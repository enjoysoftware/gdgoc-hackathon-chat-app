# チャットアプリ 設計仕様書

このドキュメントでは、Next.js と Firestore を使用したチャットアプリのデータ構造とコンポーネント構成について解説します!!!!

## 1. Firestore ドキュメントツリー構造

ユーザー情報およびメッセージ履歴を管理するための構造を採用しています。

### users (collection)
ユーザーのプロフィール情報とオンライン状態を管理します。
```text
/users (collection)
  └ {userId} (document) - Firebase Auth の UID
      ├ uid: string         (ユーザーID)
      ├ displayName: string (表示名)
      ├ photoURL: string    (アイコン画像URL)
      ├ email: string       (メールアドレス)
      ├ status: string      ("online" | "offline")
      └ updatedAt: timestamp (最終更新日時)
```

### channels (collection)
メッセージ履歴をチャンネルごとに分離して保持します。
```text
/channels (collection)
  └ {channelId} (document)
      └ /messages (sub-collection)
          └ {messageId} (document)
              ├ text: string         (本文)
              ├ senderId: string     (送信者のUID)
              ├ senderName: string   (送信時の表示名,退会時の予備表示用)
              ├ senderAvatar: string (送信時のアイコン,退会時の予備表示用)
              ├ timestamp: timestamp (送信日時)
              └ mentions: string[]   (任意: "@problem-123" などのメンションID)
```

## 2. コンポーネント構成と解説

### 1. `Chat (components/Chat.tsx)`
アプリのメインコンテナとなるコンポーネントです。
- **閲覧権限**: ゲスト（未ログイン）ユーザーでもメッセージの閲覧が可能。
- **プロフィール動的解決**: メッセージ送信時の固定情報ではなく、`users` コレクションからリアルタイムに現在の表示名とアバター画像を取得して表示します。
- **メンション解析**: `lib/utils/mentions.ts` を使用してメッセージ内の `@problem-xxx` 形式のテキストを解析し、後述する `ProblemTag` としてレンダリングします。
- **問題分析機能**: `ProblemTag` がクリックされると、その ID に紐づくメッセージ群を収集し、分析 API を呼び出してグラフデータを取得・表示します。
- **ブラッシュアップ支援**: 入力テキスト・分析状態・パネル表示フラグを一元管理し、`BrushUpPanel` および `BrushUpDetailModal` の表示を制御します。

### 2. `Sidebar (components/Sidebar.tsx)`
画面左側のナビゲーションエリアです。
- **役割**: チャンネル一覧を表示。未ログイン時はプロフィールの代わりにログインボタンを表示。
- **機能**: `next/navigation` の `useRouter` を使用した遷移、現在選択中のチャンネルのハイライト表示。

### 3. `MessageInput (components/MessageInput.tsx)`
画面下部のメッセージ入力エリアです。
- **役割**: メッセージの入力と送信。未ログイン時は入力が無効化（disabled）されます。
- **機能**: Firestore へのデータ保存、Enterキー送信、プレースホルダーの動的切り替え、「質問を分析」ボタンの表示。
- **状態管理**: 入力テキストや分析トリガーは親の `Chat` から props で受け取ります（`value`, `onChange`, `onAnalyze`, `isAnalyzing`, `onAfterSend`）。

### 4. `Login (components/Login.tsx)` / `useAuth (hooks/useAuth.ts)`
認証機能の基盤です。
- **役割**: Google ログイン/ログアウト、`users` コレクションへの自動同期。

### 5. `Firebase Config (lib/firebase.ts)`
- **役割**: Firestore (`db`) および Auth (`auth`) の初期化。

### 6. `Geminiのテキスト生成の利用(app/api/gemini-response/route.ts)`
- **役割**: Gemini APIへのリクエストのラッパー。
#### 注: Gemini APIの使用方法
Content-Typeを`application/json`とし、POSTリクエスト、JSON形式でプロンプトをリクエストボディに含めてください。動作確認用のスクリプトは`test_api.sh`にあります。  
リクエストボディ形式: `{"prompt": "プロンプトをここに入力"}`  
レスポンス: AIからの応答がストリーミング形式で返されます。


### 7. 問題分析とグラフ表示 (components/graph/)
特定のトピックや問題に関するメッセージの関連性を視覚化します。
- **`GraphPanel (components/graph/GraphPanel.tsx)`**: グラフ表示エリア全体を管理するサイドパネル/モーダル。
- **`GraphVisualization (components/graph/GraphVisualization.tsx)`**: React Flow を使用して、メッセージ間の関連（原因、結果、対策など）をノードとエッジのグラフとして描画します。
- **`GraphTimeline (components/graph/GraphTimeline.tsx)`**: 発生した事象を時系列に沿って一覧表示します。
- **`ProblemTag (components/ui/ProblemTag.tsx)`**: メッセージ内のメンションをインタラクティブなボタンとして表示する UI コンポーネント。

### 8. 質問ブラッシュアップ支援 (components/brushup/)
メッセージ送信前にAIが下書きを分析し、不足情報を補うサジェストを提示します。
- **`BrushUpPanel (components/brushup/BrushUpPanel.tsx)`**: メッセージ入力フィールドの直上に表示されるインラインパネル。分析中・エラー・結果の3状態を管理します。
- **`SuggestionCard (components/brushup/SuggestionCard.tsx)`**: サジェスト1件をカードとして表示する UI コンポーネント。クリックで入力欄にカテゴリの補足テンプレートを追記します。
- **`BrushUpDetailModal (components/brushup/BrushUpDetailModal.tsx)`**: 「詳細に分析する」ボタンで開くモーダル。下書き全文とサジェスト一覧を表示します。
- **`useBrushUp (hooks/useBrushUp.ts)`**: Gemini API（`/api/gemini-response`）へのリクエスト送信・ストリーミングレスポンスの読み取り・JSONパースを管理するカスタムフックです。

#### 型定義 (`types/brushup.ts`)
```text
BrushUpSuggestion
  ├ category: string  (短いカテゴリラベル。例: 「なぜ（目的）」「対象（どこで）」)
  └ question: string  (ユーザーへの具体的なサジェスト質問文)

BrushUpAnalysis
  └ suggestions: BrushUpSuggestion[]  (最大3件)
```

#### AI連携の仕組み
1. 「質問を分析」ボタン押下時に、下書きテキストとシステムプロンプトを結合して `/api/gemini-response` に POST します。
2. システムプロンプトでは「ベテランエンジニア」として不足情報を引き出す役割を指示しています。
3. レスポンスはストリーミング形式で受信し、全文結合後に JSON としてパースします。

## 3. 問題分析ロジック (Heuristic Analyzer)

アプリはメッセージ群から構造化された知見を抽出するヒューリスティック分析エンジンを備えています。

### 1. 分析 API (`app/api/analyze-problem/route.ts`)
クライアントからのメッセージデータを受け取り、以下の処理を順次実行してグラフ構造を生成します。

### 2. 分析エンジンのプロセス (`lib/graph/analyzer.ts`)
1. **ノード分類 (Classification)**: キーワードマッチング等により、各メッセージを「症状(Symptom)」「原因(Cause)」「仮説(Hypothesis)」「対策(Action)」等の型に分類します。
2. **エッジ推論 (Edge Inference)**: タイムスタンプ、返信関係、出現キーワードの類似性からメッセージ間の因果関係や関連性を推論します。
3. **タグ抽出 (Tag Extraction)**: 頻出する重要なキーワードをタグとして抽出します。
4. **サマリー生成**: 分析結果の全体像を要約します。

### 3. レイアウト計算 (`lib/graph/layout.ts`)
React Flow 上でノードが重ならないよう、時系列に基づいた配置（Time-based layout）を自動計算します。

## 4. ルーティング設計

Next.js の App Router を活用した動的ルーティングを採用しています。

- **`/` (app/page.tsx)**:
  - アクセス時に自動的に `/project-alpha` へリダイレクトします。
- **`/[channelId]` (app/[channelId]/page.tsx)**:
  - URL パスから `channelId` を取得し、それを `Chat` コンポーネントにプロップスとして渡します。

---

## 5. UI/UX デザイン方針

- **テーマ**: ダークモード（`#0b1426` ベース）を採用。
- **アクセシビリティ**: ログイン不要で閲覧可能にすることで、ユーザーの離脱を防止。
- **リアルタイム性**: 全てのチャンネルで即座なデータ同期を実現。
