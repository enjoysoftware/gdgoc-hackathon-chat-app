# チャットアプリ 設計仕様書

このドキュメントでは、Next.js と Firestore を使用したチャットアプリのデータ構造とコンポーネント構成について解説します。

## 1. Firestore ドキュメントツリー構造

ユーザー情報およびメッセージ履歴を管理するための構造を採用しています。

### users (collection)
ユーザーのプロフィール情報とオンライン状態を管理します。
```text
/users (collection)
  └ {userId} (document) - Firebase Auth の UID
      ├ uid: string         (ユーザーID)
      ├ displayName: string (表示名)
      ├ photoURL: string    (アイコン画像URL)
      ├ email: string       (メールアドレス)
      ├ status: string      ("online" | "offline")
      └ updatedAt: timestamp (最終更新日時)
```

### channels (collection)
メッセージ履歴をチャンネルごとに分離して保持します。
```text
/channels (collection)
  └ {channelId} (document)
      └ /messages (sub-collection)
          └ {messageId} (document)
              ├ text: string         (本文)
              ├ senderId: string     (送信者のUID)
              ├ senderName: string   (送信時の表示名,退会時の予備表示用)
              ├ senderAvatar: string (送信時のアイコン,退会時の予備表示用)
              └ timestamp: timestamp (送信日時)
```

## 2. コンポーネント構成と解説

### 1. `Chat (components/Chat.tsx)`
アプリのメインコンテナとなるコンポーネントです。
- **閲覧権限**: ゲスト（未ログイン）ユーザーでもメッセージの閲覧が可能。
- **プロフィール動的解決**: メッセージ送信時の固定情報ではなく、`users` コレクションからリアルタイムに現在の表示名とアバター画像を取得して表示します。これにより、ユーザーがプロフィールを変更すると過去のメッセージのアバターも自動的に更新されます。
- **表示ロジック**: 取得したメッセージの `senderId` と現在のユーザーの `uid` を比較し、自分か他人のメッセージかを判別してレイアウトを切り替えます。

### 2. `Sidebar (components/Sidebar.tsx)`
画面左側のナビゲーションエリアです。
- **役割**: チャンネル一覧を表示。未ログイン時はプロフィールの代わりにログインボタンを表示。
- **機能**: `next/navigation` の `useRouter` を使用した遷移、現在選択中のチャンネルのハイライト表示。

### 3. `MessageInput (components/MessageInput.tsx)`
画面下部のメッセージ入力エリアです。
- **役割**: メッセージの送信。未ログイン時は入力が無効化（disabled）されます。
- **機能**: Firestore へのデータ保存、Enterキー送信、プレースホルダーの動的切り替え。

### 4. `Login (components/Login.tsx)` / `useAuth (hooks/useAuth.ts)`
認証機能の基盤です。
- **役割**: Google ログイン/ログアウト、`users` コレクションへの自動同期。

### 5. `Firebase Config (lib/firebase.ts)`
- **役割**: Firestore (`db`) および Auth (`auth`) の初期化。

## 3. ルーティング設計

Next.js の App Router を活用した動的ルーティングを採用しています。

- **`/` (app/page.tsx)**:
  - アクセス時に自動的に `/project-alpha` へリダイレクトします。
- **`/[channelId]` (app/[channelId]/page.tsx)**:
  - URL パスから `channelId` を取得し、それを `Chat` コンポーネントにプロップスとして渡します。

---

## 4. UI/UX デザイン方針

- **テーマ**: ダークモード（`#0b1426` ベース）を採用。
- **アクセシビリティ**: ログイン不要で閲覧可能にすることで、ユーザーの離脱を防止。
- **リアルタイム性**: 全てのチャンネルで即座なデータ同期を実現。
